# docs/INDEX — карта документации

Точка входа. Прочитай этот файл первым, потом грузи нужный спек.

## Порядок чтения

1. `AGENTS.md` — правила работы в репозитории.
2. `INDEX.md` (этот файл) — карта.
3. `BACKLOG.md` — очередь задач.
4. `ARCHITECTURE.md` — архитектура системы (сервисы, брокер, потоки, деплой).
5. `LAYOUT.md` — раскладка каталогов монорепо.
6. `DEPLOYMENT.md` — docker-compose, сети, запуск локально.
7. `STACKS.md` — toolchain и layout выбранного стека.
8. `specs/<service>/<module>.md` — контракт нужного модуля.

## Файлы

| Файл | Что описывает |
|---|---|
| AGENTS.md | Правила: ветвление, что можно/нельзя, коммиты, ADR, команды по стекам |
| BACKLOG.md | Очередь задач с приоритетами и зависимостями |
| ARCHITECTURE.md | Система: сервисы, брокер, потоки данных, деплой |
| LAYOUT.md | Раскладка каталогов монорепо: сервисы, спеки, общее |
| DEPLOYMENT.md | docker-compose: сервисы, сеть, volumes, запуск локально |
| STACKS.md | Toolchain, layout и команды для Python/Go/Rust/TS |
| specs/<svc>/<mod>.md | Контракты модулей — по сервисам |
| adr/*.md | Архитектурные решения (ADR) |

> Список спеков заполни под свои сервисы/модули. Удали `specs/EXAMPLE.md`.

## Как работать со спеками

- Прочитай `ARCHITECTURE.md` перед работой с любым модулем.
- Прочитай спек модуля перед изменением кода в нём.
- Спеки живут в `docs/specs/<service>/<module>.md` — по одному на модуль.
- После реализации — обнови спек: перенеси пункт из «Что TODO» в «Что есть».
- Каждый пункт «Что есть» в спеке → должен иметь тест.
- Каждый спек следует структуре: описание / Интерфейсы / Типы / Что есть /
  Что TODO / Ограничения / Зависимости.
- Спек описывает контракт (что), не реализацию (как).
- Как соотнести модуль с каталогами выбранного стека — см. `LAYOUT.md`.

## Проверка перед коммитом

Три группы команд, все должны пройти:

- **lint** — форматирование и линтер.
- **test** — тесты.
- **build** — сборка.

Команды выбранного стека — в `AGENTS.md` → *Команды проверки по стекам* и в
`STACKS.md`. Затронуто несколько сервисов → проверить каждый.

Если любая fail — чини, не коммить.
